Installation

Client

There is almost no client installation required. The client needs to have the latest version of Internet Explorer (or pretty much any other modern web browser).

The client has to join the appropriate network, typically via WiFi.

The administration menu offers a small installer that will create desktop shortcuts.

Server

Reg3 is distributed as a single .EXE file named Reg3-<release>.exe. To install:

  • Execute that file.
  • A small popup window will ask for confirmation.
  • A command-prompt window will ask for the name of a directory name to use for the convention.
  • If that directory does not yet exist, you'll be asked whether you want to set it up. If it does exist, you'll be asked whether you want to upgrade the existing installation.

    • A fresh installation is quite boring. Before it will be useful, you must:

  • Set convention-global configuration values - convention name, start date, et cetera.
  • Install label printer drivers on servers.
  • Mark label printers as being label printers.
  • Allocate membership numbers for each server to assign.
  • Set up membership classes.
  • Set up stations, assigning a label printer to each.

    • Probably:

  • Set up an import map to import data from pre-convention sources.
  • Import pre-convention data.

    • Optionally:

  • Set up "standard" upgrades.
  • Set up metaclasses, assign classes to metaclasses, and authorize appropriate stations to issue memberships from those metaclasses.
  • Set permissions appropriately for particular stations.
  • Add custom fields.
  • Assign names to servers, stations, and label printers.
  • Set up member categories.
  • Set up automatic corrections.
  • Set up export maps.
  • Renumber memberships if pre-convention data does not include membership numbers.

    • You might be able to use Administration Synchronize Export Selected and Import to copy settings from another convention.

    Conventions each get their own directory, in Documents\Reg3\$CON.

    Conventions share Node.js installations, in per-version directories. That way a particular Reg3 instance can use its favorite Node.js version. Node.js is installed in Documents\Reg3\node-vXX.YY.Z-win-x86.

    Conventions do not share Reg3 itself. This keeps the convention data with a matched copy of the program.
