Filters

The filter user interface appears in a number of places to allow you to select records based on arbitrary criteria.

It defaults to matching all records. You can pick operators off of a pull-down list to add criteria. All operators other than "All" require one or more parameters, each of which can be another expression.

The record matches if the result is "truthy" - non-empty, non-zero, true - and does not match if the result is "falsey".
Operator Arguments Description
All None Always true
All of Any number True if none of its arguments are falsey.
Technical note: starts with true, evaluates arguments until it reaches a falsey value or the end, and returns the last value processed.
Any of Any number True if any of its arguments are truthy.
Technical note: starts with false, evaluates arguments until it reaches a truthy value or the end, and returns the last value processed.
Not One True if its argument is falsey, and false if it its argument is truthy.
If Three If its first argument is truthy, returns its second argument. If the first argument is falsey, returns its third argument.
Technical note: either the second or third argument is evaluated, not both.
Equal Two Returns true if its arguments are the same, and false if not.
Technical note: although it is not an error to compare unlike types - text and a number, say - the comparison rules are tricky and it is best to compare only like types.
Match One, constant Splits the argument into individual words. Scans all of the fields in the record; returns true if all of the words are found in the record. This is the same matching mechanism used for "Previously registered" lookups.
Class any of Any number, class names Arguments are names of defined member classes. Returns true if the member class is any of the classes listed.
Category any of Any number, category names Arguments are names of defined member categories. Returns true if the member is in any of the categories listed.
Note: use "All of" combined with multiple "Category any of" operators to match records that are in all of the categories listed.
Text One, constant Constant textual value, typically for use in an "Equal" comparison.
Number One, constant Constant number, typically for use in an "Equal" comparison.
Field One, field name Retrieves the value from the specified field in the current record.
