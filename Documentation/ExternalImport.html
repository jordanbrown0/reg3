External Import

Import data from an external file.

  • The file to import. Click Browse, then select the file to import. Ensure that the file type matches the type expected by the import mapping specified below.
  • What import mapping to use. Ensure that the file type expected by the import mapping matches the type of the file selected above.
  • Controls what happens when the imported data includes records that are already present in the system:
      Keep existing record The existing record in the registration system is kept. The record from the import file is ignored. (Requires that the import mapping specify a unique identifier field.)
      Replace existing record The record from the import file is used; the existing record in the registration system is discarded. (Synchronization data is retained so the newly imported record will propagate to other servers.) (Requires that the import mapping specify a unique identifier field.)
      Update existing record Records from the import file that do not exist in the registration system are ignored. Existing records are updated, field by field, with the fields present in the import file. (Requires that the import mapping specify a unique identifier field.)
      Zap all existing records DISCARDS all records currently existing in the registration system; imports new records from the import file. CAUTION!
      Add duplicate Adds all records from the import file. If they already exist in the registration system, this will add duplicate records. You almost never want this option.
